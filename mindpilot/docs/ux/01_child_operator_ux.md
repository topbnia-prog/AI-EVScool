# Child Operator UX

## UX Goal

The child should feel like an operator in a serious system, not a student in a school portal.

## Visual Tone

- Technical HUD
- Dark blue base
- Orange accent
- Thin lines
- Monospace headings
- Inter or similar for body text
- No cartoon mascot
- No public leaderboard
- No childish visual overload

## Core Screens

1. Operator Dashboard
   - Current mission
   - Streak
   - Rank
   - XP energy
   - Mission path
   - Metric snapshot

2. Mission Screen
   - Six-step mission progress
   - Hook
   - Concept
   - Challenge
   - Simulation
   - Reflection
   - Continue controls

3. Mentor Chat
   - Full-screen on mobile
   - Current mission context visible
   - Mentor asks questions
   - Mentor does not give final answers

4. Pilot Profile
   - Six metrics
   - Personal growth
   - Achievements
   - Best streak
   - No comparison with other children

## Streak Policy

Streak is soft:

- One meaningful action per day counts.
- One weekly Streak Freeze is granted.
- Freeze does not stack.
- If streak breaks, no shame message.
- Recovery message: "Start a new series today. The first 3 days give double XP."

## Game Moments

The interface is calm most of the time. Animation appears only on:

- LevelUpModal
- AchievementToast
- StreakFlame

All other transitions should be subtle.
