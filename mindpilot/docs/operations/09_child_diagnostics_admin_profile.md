# Child Diagnostics And Admin Profile

## Purpose

Before the first lesson MindPilot should not treat every child the same. The system needs a small, respectful profile that helps the mentor and admin choose the right tone, examples, pace, and first challenge.

This is not a psychological diagnosis. It is a product and learning-context profile.

## Parent Registration Survey

The parent sees the child from the outside. During registration we collect:

- how the child usually enters a new topic
- what blocks or frustrates the child
- what the parent wants MindPilot to develop first
- whether the child needs a calm, practical, challenge-based, or story-based approach

The parent answers should shape:

- first session tone
- parent dashboard summary
- admin child profile
- mentor personalization rules

## Child MindScan

Before the first mission the child answers a short MindScan:

- current experience with AI
- preferred pace
- favorite challenge style
- preferred help style when stuck
- interest topics
- one personal learning goal

MindScan must feel like setting up a game profile, not like a school test.

## Admin Child Profile

The admin profile should summarize:

- short headline about the child as an operator
- recommended approach
- motivators
- risks
- mentor tone
- first session plan
- parent insight
- MindScan answers
- current metrics and safety events

This gives the admin a useful “picture of the child” before any manual intervention.

## Safety Rule

Do not collect sensitive personal data in these surveys. Avoid medical labels, diagnoses, school names, photos, location, or private family details. Keep answers practical and learning-focused.
